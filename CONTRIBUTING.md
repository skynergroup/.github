# Contributing to Skyner Group Projects

## Getting Started

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Open a pull request

## Pull Request Process

- All PRs require at least 1 approving review from a code owner
- All conversations must be resolved before merging
- We use squash merging — your PR title becomes the commit message

## Code Standards

- Follow existing code conventions in the repo
- Run linters before committing
- Write tests for new functionality

## Questions?

Open a discussion in the relevant repository.
